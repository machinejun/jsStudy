1. 운영체제
: 컴퓨터 시스템의 자원들을 효율적으로 관리 => 자원의 스케줄링 기능
: 사용자가 컴퓨터를 편리하고 효과적으로 사용할 수 있도록 환경을 제공하는 여러 프로그램의 모임
: 프로세스, 기억장치, 주변 장치, 파일 등을 관리하는 기능을 수행

ⓐ 목적
- 처리 능력 : 일정 시간 내에 시스템이 처리하는 일의 양
- 반환 시간 : 시스템에 작업을 의뢰한 시간 부터 처리가 완료될 때까지 걸린 시간
- 사용 가능도 : 시스템을 사용할 필요가 있을 때 즉시 사용 가능한 정도
- 신뢰도 : 시스템이 주어진 문제를 정확하게 해결하는 정도

※ 프로세스 = 작업(Job) = 태스크(Task)
: 주기억장치에 저장된 프로그램 = 실행중인 프로그램

ⓑ 종류
: Window, Unix, Linux, MacOs, Ms-Dos 등

- Unix
: AT&T 벨 연구소, MIT, General Electronic 이 공동 개발한 운영체제
: 대화식 운영체제
: 오픈소스 기반
: C 언어로 작성되어서 이식성이 높으며, 프로세스간의 호환석이 높다.
: 다중 사용자, 다중 작업을 지원
: 트리구조 파일 관리 시스템

- Linux
: 리누스 토발즈가 UNIX를 기반으로 개발한 운영체제
: 오픈소스 기반
: unix와 완벽 호환

- MacOs
: 애플 사가 Unix를 기반으로 개발

- Android 
: 구글사에서 개발한 리눅스 커널 기반의 개방형 모바일 운영체제
: 자바 & 코틀린으로 애플리케이션을 작성한다.

- Ios
: 애플사에서 개발한 유닉스 기반 모바일 운영체제

2. 운영체제 제어
- CLI : 사용자가 직접 명령어를 입력, 컴퓨터에 명령을 내리는 방식
- GUI : 마우스로 화면을 클릭하여 그래픽 위주로 컴퓨터를 제어하는 방식

3. 리눅스/유닉스 파일 접근 권한 관리
- 유형
User = u : 사용자
Group = g : 같은 그룹에 속해있는 모든 사용자 모임
other = o : 그 밖의 사용자

- 연산자
+ : add
- : remove
= : set

- 타입
r : read
w : write
x : excute

- 타입 숫자
r : 4
w : 2
x : 1

chmod g-r test.txt => g 그룹에 read 권한을 제거해도
chmod g+rw test.txt => g 그룹에 read, write 권한 추가
chmode 777 text.txt => u = rwx, g = rwx, o = rwx


4. 기억장치 전략
- 보조기억장치의 프로그램의 데이터를 주기억장치에 적재시키는 시기, 위치등을 지정하여 한정된 주기억장치의 공간을 효율적으로 사용
- 종류 = 반입, 배치, 교체

ⓐ 반입 
: 메모리로 적제 시기 결정 = when
: 요구 반입 기법, 예상 반입 기법

ⓑ 배치 
: 메모리 적재 위치 결정 = where
: 최초 적합 = 첫번째 분할에 할당
: 최적 적합 = 크기가 가장 비슷한 공간에 할당
: 최악 적합 = 가장 큰 공간에 할당

ⓒ 할당 
: 메모리 적재 방법 결졍 = how
: 연속 할당 기법, 분산 할당 기법

ⓓ 교체 
: 메모리 교체 대상 결정 = who
: Swap In/Out, FIFO, LRU, LFU ....

5. 프로세스 관리

- 상태 전이 ★
① 생성 상태 : 프로세스가 생성된 상태
② 준비 상태 : CPU 할당 받을 수 있는 상태
③ 실행 상태 : CPU에 할당 받아 동작하고 있는 상태
④ 대기 상태 : 입출력 처리 등으로 CPU를 양도하고 입출력등 처리가 완료되어 까지 대기 리스트에서 기다리는 상태
⑤ 완료 상태 : 주어진 시간 내 완전히 수행 후 종료

※ 디스패치 : 준비 리스트에서 프로세스를 선정하여 CPU에 할당( 준비 -> 실행 )
※ timer runout : 지정 시간 초과시 CPU를 반납하고 준비 상태로 전의 ( 실행 <- 준비 )
※ 블록 : 입출력 등이 발생하면 스스로 CPU를 반납하고 대기상태로 전의 ( 실행 -> 대기 )
※ wake up : 입출력이 종료되면 wait&signal 등에 의해서 실행상태로 전의 ( 대기 -> 실행 )

7. 가상화
- 가상화 : 물리적인 리소스들을 사용자에게 하나로 보이게 하거나 하나의 물리적인 리소스를 여러개로 보이게 하는 기술
- 종류 : 플랫폼, 리소스 가상화
- 기술
▷ 컴퓨터 가상화
▷ 스토리지 가상화
▷ I/O 가상화
▷ 컨테이너
▷ 분산처리기술

7. 클라우딩
- 인터넷을 통해 가상화된 컴퓨터 리소스를 제공하고, 정보를 클라우드에 연결된 다른 컴퓨터로 처리하는 기술
- 유형
▷ 인프라형 (IaaS) : 시스템 자원을 클라우드로 제공, 컴퓨터 운영체제나 애플리케이션 등 소프트웨어 탑재 및 실행
▷ 플랫폼형 (PaaS) : 애플리케이션을 개발, 실행, 관리할 수 있게 하는 플랫폼을 제공
▷ 소프트웨어형 (SaaS) : 소프트웨어 및 관련 데이터는 중앙에 호스팅 되고 사용자는 
                        클라이언트를 통해 접속하여 소프트웨어를 서비스 형태로 이용하는 서비스
