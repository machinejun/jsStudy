네트워크

2. 프로토콜
ⓐ 기본요소
- 구문 : 전송하고자 하는 데이터의 형식, 부호화, 신호레벨을 규정
- 의미 : 두 기기간의 효율적 정보 전송을 위해 제어 정보를 규정
- 시간 : 통신 속도, 메세지 순서 제어등을 제어등을

ⓑ 기능
- 단편화 : 송신 측에서 데이터 전송에 알맞은 일정 크기의 작은 블록으로 데이터를 자르는 작업
- 재결합 : 단편화된 블록을 원래 형태로 복원
- 캡슐화 : 데이터를 오류없이 정확하게 전송하기 위해 캡슐화
- 흐름제어 : 수신측의 처리 능력에 따라 송신 측에서 송신하는 데이터의 전송량이나 전송 속도를 조절
- 오류제어 : 전송중에 발생하는 오류를 검출하고 데이터 전송에 분실, 파손에 대비
- 동기화 : 송수신 측이 같은 상태를 유지하도록 타이밍을 맞추는 작업
- 순서제어 : 전송되는 데이터 블록에 전송순서를 부여하여
- 주소지정 : 데이터가 목적지 까지 정확하게 전송되도록 이름, 주소, 경로를 부여하여
- 다중화 : 한개의 통신 회선을 여러 가입자들이 동시에 사용하도록 하는 기능
- 경로제어 : 송수신 측 간의 송신 경로중 최적의 교환 경로를 지정

3. OSI 참조 모델
: ISO에서 제안한 통신 규약 프로토콜

ⓐ application layer
- 응용 프로세스와 직접 관계하여 일반적인 응용 서비스를 수행하는 역할

- 프로토콜 ★
> SOAP : HTTP, HTTPS
> FTP
> SMTP
> POP
> IMAP
> telnet 

ⓑ presentation layer
- 암복화등 데이터의 표현

ⓒ session layer
- 연결설정을 관리

ⓓ transport layer
- 양 종단간의 신뢰성있는 정보를 전달
- 프로토콜
> TCP 
: 양방향 연결, 가상회선 연결, 패킷 단위
 
> UDP
: 비 연결형 서비스 제공
: TCP에 비해서 단순한 헤더 구조

> RTCP
: RTP패킷 전송 품질을 제어하기 위한 프로토콜
: 세션에 참여한 각 참여자들에거 주기적으로 제어 정보를 전송

ⓔ network layer
- 라우터 : 목적지 까지 빠르게 데이터를 전송하는 기능 => L3장비
- 프로토콜 
> IP : 송수시간의 패킷 단위로 데이터를 교환하는 통신 프로토콜
> ARP : IP주소를 Mac 주소로 변환
> RARP : Mac주소를 IP 주소로 변환
> ICMP : IP패킷을 처리할 때 발생하는 문제를 알려주는 프로토콜
> IGMP : 호스트 컴퓨터와 인접한 라우터가 멀티캐스트 그룹 멤버십을 구상하는되 사용되는 프로토콜
> 라우팅 프로토콜 : 데이터 전송을 위해 목적지까지 갈 수 있는 여러 경로 중 최적의 경로를 선정해주는 통신규약
  >> RIP : 거리 벡터알고리즘에 기초하여 개발된 프로토콜(벨만-포드 알고리즘)
  >> OSPF : TCP/IP 네트워크에서 RIP 단점 개선 --> 자신을 기준으로 상대 알고리즘을 적용하여 최단경로를 찾은 알고리즘 
  >> BGP : AS 상호간 경로 정보를 교환하기 위한 라우팅 프로토콜

- IPv4
: 인터넷에 연결된 모든 컴퓨터 자원을 구분하기 위한 고유한 주소
: 8비트씩 4부분, 32비트로 구성
: A ~ E 클래스 까지 5단계로 구성되어 있음 but E클래스는 실험적 주소이기에 공용되지 않음
: 유니캐스트, 멀티캐스트, 브로드 캐스트

A = 국가, 대형 통신망에 사용(0 ~ 127) but 0, 127번은 예약된 주소이므로 1 ~ 126으로 시작된다.
B = 중대형 통신망에 사용(128 ~ 191)
C = 소규모 통신망에 사용(192 ~ 223)
D = 멀티테스크용으로 사용(224 ~ 239)

-IPv6
: 128비트의 긴 주소를 사용
: IPv4에 비해 자료 전송 속도가 빠르다
: 인증성, 기밀성, 데이터 무결성의 지원
: 유니캐스트, 멀티캐스트, 애니캐스트

※ 멀티캐스트 : 인터넷에서 같은 내용의 데이터를 여러 명의 특정한 그룹의 수신자들에게 동시 전송 할 수 있는 프로토콜
※ 유니캐스트 : 고유 주소로 식별된 하나의 네트워크 목적지에 1:1로 트래픽 또는 메시지를 전송하는 프로토콜
※ 브로드캐스트 : 하나의 송신자가 같은 서브 네트워크상의 모든 수신자에게 데이터를 전송하는 프로토콜
※ 애니캐스트 : 하나의 수신 주소로 식별되는 다수의 노드로 데어터그램 전송

ⓕ data link layer
- 기능 : 흐름제어, 프레임 동기화, 오류제어, 순서제어 
- 장비 : 브리지, 스위치 => L2 장비

ⓖ physical layer 
- 장비 = 리피터, (멍청이)허브


★ 캡슐화
H / Data : Segment 
H / Seg : Packet 
H / packet : Frame

네트워크 전달방식
- 패킷 스위칭 : 작은 블록(패킷)으로 데이터를 전송하며 데이터를 전송하는 동안만 네트워크 자원을 사용하도록 하는 방식
- ATM : 비동기 전송모드라고 하는 광대역 전송에 쓰이는 스위칭 기법
